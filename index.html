Publishing to GitHub Pages
This file is a single self-contained HTML document — no server, no dependencies, no build step. Follow these steps to publish it as a private or public URL you can share.
1
Create a new private GitHub repository
Go to github.com → New repository. Name it something like atg-intel. Set visibility to Private if you want restricted access, or Public for a shareable link. Initialize with a README.
2
Upload index.html
Click "Add file" → "Upload files" in your new repo. Drag the index.html file. Commit directly to main. That's it — no other files needed.
3
Enable GitHub Pages
Go to Settings → Pages → Source: Deploy from branch → Branch: main → Folder: / (root). Click Save. GitHub will show you a URL like https://yourusername.github.io/atg-intel/
4
Access control (optional)
For a truly private URL, keep the repo private and use GitHub Pages with authentication — or simply share the file directly via email/Dropbox. The HTML is fully self-contained and works offline.
# Alternative: CLI approach
git init atg-intel && cd atg-intel
cp /path/to/index.html .
git add . && git commit -m "ATG sales intelligence dashboard"
gh repo create atg-intel --private --push --source=.
# Then enable Pages in GitHub.com settings
5
Updating the dashboard
When you receive new LinkedIn profiles or want to update intelligence, re-upload the file to replace index.html in the repo. GitHub Pages deploys in under 2 minutes. All data is embedded directly in the HTML — no database or API needed.
